# Wireshark Network Traffic Analysis

This project demonstrates hands-on network traffic analysis using **Wireshark** as part of my SOC Analyst learning journey.

## Project Overview

The objective of this project was to capture, inspect, and analyze network traffic to understand common protocols, identify communication patterns, and practice basic threat detection techniques.

## Tasks Performed

* Captured live network traffic using Wireshark.
* Analyzed TCP, UDP, HTTP, DNS, and ICMP packets.
* Identified the local IP address and external network connections.
* Investigated HTTP requests and responses.
* Performed DNS query analysis and resolved domain names.
* Captured and analyzed ICMP (Ping) traffic.
* Examined HTTP credentials to understand why unencrypted protocols are insecure.
* Inspected network traffic for suspicious activities and unusual communication patterns.

## Skills Gained

* Network Packet Analysis
* Protocol Analysis (TCP, UDP, HTTP, DNS, ICMP)
* DNS Investigation
* Traffic Filtering using Wireshark
* Network Troubleshooting
* Basic Threat Hunting
* SOC Analyst Fundamentals

## Tools Used

* Wireshark
* Windows
* Google Chrome
* Command Prompt (Ping)

## Key Findings

* TCP generated most of the captured traffic.
* Successfully analyzed HTTP requests and DNS lookups.
* Measured ICMP response times using ping.
* Observed that HTTP transmits credentials in plain text and is not secure.
* No suspicious or malicious network activity was detected during the analysis.

## Learning Outcome

This project strengthened my understanding of packet analysis, network protocols, and traffic investigation. It provided practical experience with Wireshark and improved my skills in identifying normal network behavior, an essential capability for a SOC Analyst.
